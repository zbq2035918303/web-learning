📄 HTML 文档结构

💡 每个 HTML 文件都需要这个基本骨架，浏览器按此结构解析页面。
<!DOCTYPE html>                    <!-- 声明 HTML5 文档类型 -->
<html lang="zh-CN">              <!-- 根元素，lang 设置语言 -->
  <head>                          <!-- 不显示在页面，存放元数据 -->
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>页面标题</title>
    <link rel="stylesheet" href="style.css">
  </head>
  <body>                          <!-- 页面可见内容放这里 -->
    <h1>Hello, World!</h1>
    <script src="main.js"></script>  <!-- JS 放 body 末尾 -->
  </body>
</html>

🧠 Head 标签

标签说明示例
<title>页面标题（显示在浏览器标签页）<title>我的网站</title>
<link>外部资源链接（样式表、图标等）<link rel="stylesheet" href="style.css">
<script>JavaScript 脚本引入或内嵌<script src="app.js"></script>
<style>内嵌 CSS 样式<style>body{margin:0}</style>
<base>设置相对链接的基础 URL<base href="https://example.com/">
<noscript>JS 不可用时的降级内容<noscript>请启用JS</noscript>
 
<link rel="icon" href="favicon.ico">                <!-- 网页图标 -->
<link rel="preload" href="font.woff2" as="font">   <!-- 预加载资源 -->
<link rel="canonical" href="https://example.com"><!-- 规范 URL -->
<script src="app.js" defer></script>              <!-- defer：DOM 解析完再执行 -->
<script src="lib.js" async></script>              <!-- async：下载完立即执行 -->

🔖 Meta 元数据

Meta 标签说明
<meta charset="UTF-8">字符编码，必须设置，防止乱码
<meta name="viewport" content="width=device-width, initial-scale=1.0">移动端响应式必备
<meta name="description" content="...">页面描述，影响 SEO 搜索结果摘要
<meta name="keywords" content="...">关键词（现代搜索引擎参考价值低）
<meta name="author" content="...">作者信息
<meta name="robots" content="index,follow">控制搜索引擎爬取行为
<meta http-equiv="refresh" content="5;url=...">5秒后跳转
<meta property="og:title" content="...">Open Graph：社交分享时的标题
<meta property="og:image" content="...">Open Graph：社交分享时的图片
 
<!-- SEO 基础三件套 -->
<meta charset="UTF-8">
<meta name="viewport" content="width=device-width, initial-scale=1.0">
<meta name="description" content="页面描述，150字以内">

<!-- Open Graph（社交分享）-->
<meta property="og:title"       content="标题">
<meta property="og:description"  content="描述">
<meta property="og:image"       content="https://example.com/img.jpg">
<meta property="og:url"         content="https://example.com">

♿ 无障碍 Accessibility / ARIA

💡 良好的无障碍设计让屏幕阅读器用户、键盘用户都能正常使用页面，也有助于 SEO。
属性说明
alt（img）图片描述，屏幕阅读器必读
label for=id将标签绑定到表单控件
aria-label为元素提供可访问名称（无可见文字时用）
aria-labelledby指向另一个元素的 id 作为名称
aria-describedby指向描述文字的元素 id
aria-hidden="true"对屏幕阅读器隐藏（装饰性图标）
aria-expanded折叠/展开状态（true/false）
aria-current="page"当前导航项
aria-live动态内容更新提醒（polite/assertive）
role指定元素角色（button/dialog/alert 等）
tabindex="0"使非交互元素可获得键盘焦点
tabindex="-1"从 Tab 顺序移除，但可用 JS 聚焦

  <svg aria-hidden="true">...</svg>
</button>

<!-- 无障碍导航 -->
<nav aria-label="主导航">
  <a href="/" aria-current="page">首页</a>
</nav>

<!-- 动态区域提醒 -->
<div aria-live="polite" aria-atomic="true">
